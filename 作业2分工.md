## 二、6人分工细化方案（以实际任务为依据）

| 角色                        | 主要职责                                                     | 关键产出                                                 | 工作量占比 |
| --------------------------- | ------------------------------------------------------------ | -------------------------------------------------------- | ---------- |
| 1. 数据同步与预处理负责人   | 安装DataX，配置reader/writer，替换MySQL驱动，解决index保留字问题，同步2–5月数据；对同步后的数据进行初步清洗（去重、缺失值处理），确保后续分析可用 | 同步成功截图、本地数据库表、清洗后数据文件               | 18%        |
| 2. 地域与交易额分析负责人   | 分析不同国家、大洲的总交易金额，绘制柱状图、地图；解读结果（如美国市场主导、北美洲强势等） | 分析代码、图表、分析结论（报告第3–5页）                  | 18%        |
| 3. 流量来源与媒介分析负责人 | 统计不同流量来源（direct, organic, referral, cpc, cpm等）的会话数、交易额、退出率；分析投资价值（如youtube流量大但转化低，dfa流量小但价值高） | 分析代码、图表、分析结论（报告第5–6页 + HW2报告第4–7页） | 18%        |
| 4. 用户行为与设备分析负责人 | 分析操作系统、浏览器、设备类型分布；新老用户行为对比（页面浏览量、停留时间）；绘制饼图/条形图 | 分析代码、图表、分析结论（HW2报告第3–4页、第11页）       | 18%        |
| 5. 商品与用户购买分析负责人 | 统计top10销售额商品、top10销量商品、top10消费用户；对比销售额与销量的关系 | 分析代码、图表、分析结论（11组报告第7–8页）              | 14%        |
| 6. 报告整合与可视化负责人   | 统一所有图表风格，整理分析代码为完整.ipynb文件，撰写实验报告（包含同步截图、分析过程、图表说明、结论），检查格式 | 完整.ipynb文件、实验报告（PDF）                          | 14%        |

## 三、任务连贯性说明

1. 数据流清晰
   1. 角色1完成同步与清洗 → 角色2–5直接使用清洗后的数据进行分析，互不冲突。
2. 并行分析高效
   1. 角色2（地域）、角色3（流量来源）、角色4（用户设备）、角色5（商品购买）可以同时开展工作，彼此独立，数据来源一致。
3. 整合阶段顺畅
   1. 角色6在分析过程中即可收集各角色的图表和结论，最后统一润色、排版，确保报告风格一致。
4. 工作量平衡
   1. 角色1、2、3、4均为18%左右，承担核心分析工作；角色5、6稍轻，但负责更细致的整合与报告撰写，整体差异控制在合理范围内。

## 四、建议协作流程

1. 第1天：角色1完成数据同步与清洗，并提供清洗后的数据给全员。
2. 第2–3天：角色2、3、4、5并行分析，角色6同步收集各模块图表和代码片段，开始搭建`.ipynb`框架。
3. 第4天：角色6整合所有内容，生成完整`.ipynb`和实验报告草稿，全员核对数据准确性和结论。
4. 第5天：最终修改、统一格式、提交。

## 五、与示例报告的对齐

- 角色2对应11组报告中的“统计不同国家的交易金额”“大洲分析”部分。
- 角色3对应11组报告中的“统计不同流量来源媒介的总交易额”以及HW2报告中的“用户流量来源分析”。
- 角色4对应HW2报告中的“用户设备观察”“新用户与老用户行为对比”。
- 角色5对应11组报告中的“统计总交易金额前10的用户”“统计top10销售额最高商品”“统计top10销量最高商品”。
- 角色6负责将以上所有内容整合成两份报告呈现的形式，并补充必要的可视化优化。
